Executive Summary ¶

This PR extracts the /proc/net scanner's per-port heuristic stack out of an inlined loop in ForwardPorts into a unit-tested portStateTracker state machine, behind injectable tracker.Tracker and iptablesRunner seams. It also wires a test:unit:guestagent target into test:unit, closing a real CI gap where the guestagent module's tests — including those added in the superseded #10262 — never ran.

The extraction is the right call: it turns "fix one race, expose another" into a directly testable state machine, the portState doc comment is an unusually clear spec, and the test suite walks every transition with fakes. All three reviewers agree the structure is sound and found no Critical issues.

Two Important issues remain, and both sit in the cleanup/retry paths that are net-new in this PR. I1 (regression, confirmed against the merge-base): retireDisappeared now deletes the loopback DNAT rule even when tracker.Remove returned an error — the pre-PR loop did continue and kept the rule. A failed host-switch unexpose therefore leaves a stale proxy with no DNAT, and the reappearance path then misclassifies the port as engine-managed, so a --network=host loopback listener stays unreachable until RD restart. I2 (gap): retryAppend's engine-takeover branch releases tracker ownership without issuing the idempotency-gated Delete, so if the prior failed Append had actually committed its rule, that rule shadows the engine's chain — the exact bug this PR exists to fix. Both fixes are a few lines and make the paths symmetric with code that already exists elsewhere in the same file.

// Remove a single entry from the port storage and calls the
// /services/forwarder/unexpose endpoint to remove the forwarded port mappings.
func (a *APITracker) Remove(containerID string) error {
	portMap := a.portStorage.get(containerID)
	defer a.portStorage.remove(containerID)

	var errs []error

	for portProto, portBindings := range portMap {
		for _, portBinding := range portBindings {
			// The unexpose API only supports IPv4
			ipv4, err := isIPv4(portBinding.HostIP)
			if !ipv4 || err != nil {
				log.Errorf("did not receive IPv4 for HostIP: %s", portBinding.HostIP)
				continue
			}

			log.Debugf("unexposing the following port binding: %+v", portBinding)

			err = a.apiForwarder.Unexpose(
				&types.UnexposeRequest{
					Local:    ipPortBuilder(a.determineHostIP(portBinding.HostIP), portBinding.HostPort),
					Protocol: types.TransportProtocol(strings.ToLower(portProto.Proto())),
				})
			if err != nil {
				errs = append(errs,
					fmt.Errorf("unexposing %+v failed: %w", portBinding, err))

				continue
			}
		}
	}

	if len(portMap) != 0 {
		portMapping := guestagentTypes.PortMapping{
			Remove: true,
			Ports:  portMap,
		}
		log.Debugf("forwarding to wsl-proxy to remove port mapping: %+v", portMapping)
		err := a.wslProxyForwarder.Send(portMapping)
		if err != nil {
			return fmt.Errorf("sending port mappings to wsl proxy error: %w", err)
		}
	}

	if len(errs) != 0 {
		return fmt.Errorf("%w: %+v", forwarder.ErrUnexposeAPI, errs)
	}

	return nil
}

This branch fires when a port is in appendFailed=true state and the engine chain has appeared since the failed Append. It releases the tracker entry but never issues an iptables Delete — it assumes appendFailed=true implies no procnet rule is installed.

That assumption contradicts the code's own idempotency design. applyLoopbackIPtablesRule's docstring (scanner_linux.go:140-143) states the Append idempotency probe exists "because a previous tick's iptables call committed the rule but returned non-zero." When that happens, appendFailed is true and the rule exists. retryAppend then marks the port delegated; retireDisappeared later drops a delegated port without a Delete (:267-274), so the rogue -A PREROUTING ... -j DNAT --to-destination 127.0.0.1:<port> rule shadows CNI-HOSTPORT-DNAT until RD restarts — precisely the bug this PR exists to fix.

A port whose engine-chain probe keeps failing (persistent xtables lock contention, fork/exec failure) never reaches pst.added, so exposeNew re-runs and logs at Error every ~3 s indefinitely. The PR deliberately added the addErrorLogged log-once mechanism for the parallel tracker.Add retry-storm (:176-181), and retryAppend's own engine-probe failure logs at Debug (:242) — so this path is an inconsistent gap in that same effort.

Fix: route this through the addErrorLogged log-once bookkeeping (loud first error, quiet thereafter, cleared when the port is added or disappears), or drop it to Debug to match retryAppend.

S2. fakeIptables does not model installed-rule state, so the commit-but-errored interaction is untested Claude¶

fakeIptables.ApplyLoopbackRule records the call and returns the injected error but never tracks whether a rule is "installed." TestAppendRetryReleasesOwnershipWhenEngineChainAppears therefore exercises only the lock-contention shape of appendFailed (no rule installed) — it cannot reach the commit-but-errored shape that I2 describes.

Fix: when fixing I2, give the fake a rule-presence model (Append sets it, Delete clears it, the idempotency probe reads it) and add a regression test asserting the engine-takeover branch issues a Delete when a rule was committed.

S3. No regression test for a failed Remove followed by a "proxy already running" reappearance Codex¶

TestCleanupReleasesAddedOnRemoveFailure lets the next tracker.Add succeed, so it misses the realistic shape I1 describes: a failed unexpose leaves a stale proxy, and the reappearance tracker.Add returns portAlreadyExposedSubstring with an empty tracker.Get.

Fix: when fixing I1, add a test where Remove returns an unexpose error, the listener reappears, the next Add returns portAlreadyExposedSubstring, and tracker.Get(syntheticID) is empty — asserting the loopback DNAT is not deleted (or is reinstalled) so the port stays reachable.

Design Observations ¶

Strengths ¶

  • (in-scope) State-machine extraction — pulling the per-port logic into portStateTracker behind injectable tracker.Tracker and iptablesRunner seams turns three rounds of "fix one race, expose another" into a directly unit-testable state machine. The portState doc comment (port_state_tracker_linux.go:28-69) is an unusually clear state-machine spec. Claude Codex Gemini
  • (in-scope) Idempotency via iptables --check — applyLoopbackIPtablesRule probes preroutingHasLoopbackRule before every Append and Delete instead of trusting in-memory state, which prevents both duplicate PREROUTING rules and spurious error logs when rules are cleaned up out of band. Gemini
  • (in-scope) Transient-vs-absent precision — isIptablesRuleAbsent distinguishes a steady-state "rule/chain not present" from transient xtables lock contention, so a probe failure defers the port rather than being misread as "not engine-managed" and slipping a shadow DNAT through. Gemini
  • (in-scope) TestPortAlreadyExposedSubstringIsReachable pins the "proxy already running" substring against the real forwarder → APITracker wrap chain — the delegation logic depends on a substring that would otherwise drift unnoticed on an upstream rename. Claude
  • (in-scope) CI gap closed — wiring test:unit:guestagent into test:unit means the guestagent module's tests (including those from the superseded #10262) finally run; the test job runs on ubuntu-latest, so the _linux-tagged files actually compile and execute. Claude

Acknowledged Limitations ¶

  • wsl-proxy failure not retried on the resume-ownership path — port_state_tracker_linux.go:54-59, :208-213. The resume path installs the iptables rule the partial-failure path skipped but does not re-send the port mapping to wsl-proxy, because Tracker exposes no idempotent re-notify hook. Documented in the portState doc comment.
  • Undeleted PREROUTING rule persists until RD restart — port_state_tracker_linux.go:282-283. The cleanup path drops the local marker unconditionally (retaining it deadlocks Add on reappearance); a stranded rule self-cleans on restart. (I1 and I2 are distinct: they cover deleting the rule when it should be kept, and keeping it when it should be deleted.)
  • Port-range iptables syntax not matched — scanner_linux.go:233-243. dpts:LO:HI and multiport-embedded ranges are intentionally not matched; shipped engines emit per-port DNAT rules, and the tcpRangeChain / multiportEmbeddedRange fixtures lock the behavior.
  • Engine chain list closed to CNI-HOSTPORT-DNAT and DOCKER — scanner_linux.go:190-194. Documented, with an explicit "revisit when adding a third engine" note.
  • English-locale stderr parsing — scanner_linux.go:305-316. isIptablesRuleAbsent matches exact English iptables phrases; safe within RD's controlled Lima/WSL utility VMs.

Review Process Notes ¶

Skill improvements ¶

  • When a code path creates or deletes one resource conditioned on the success of an operation on a different resource, check the partial-failure matrix: can the conditioning operation fail in a way that leaves its own state half-applied, so its return value no longer predicts whether the dependent resource should exist? Flag paths where an error from operation A is treated as "A did nothing" when A actually mutates shared state before returning that error, or where success of A is treated as "the dependent resource is now consistent" when A only partially completed. This is the shape behind a whole class of cleanup/retry bugs and is not obvious from reading either side in isolation.

Repo context updates ¶

  • [repo] tracker.APITracker mutates shared state before returning errors, so its error returns do not cleanly map to "nothing happened." Remove clears portStorage via a leading defer even when the host-switch unexpose API fails — a non-nil error means "storage cleared, proxy may still be bound." Add runs portStorage.add and apiForwarder.Expose before wslProxyForwarder.Send, so a non-nil error can mean "storage populated and proxy exposed, wsl-proxy not notified." When reviewing guestagent port-forwarding code (procnet, iptables, callers of tracker.Tracker), check that any caller installing or removing a downstream artifact (an iptables PREROUTING rule, a local marker) keyed off Add/Remove success accounts for these partial-failure states.
