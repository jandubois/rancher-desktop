Executive Summary ¶

The PR drops procnet's PREROUTING DNAT writes in favor of a userspace TCP/UDP proxy bound on the tap-interface IP, deleting the 665+1187-line engine-chain probing surface #10280 carried. The architectural shift is sound. One critical issue: the new forwarder's own tap-IP listener appears in /proc/net/tcp, so the scanner never observes the original loopback listener "disappear" and never unpublishes it — every --network=host loopback container leaves a stuck forwarder, host-switch entry, and accept goroutine behind on exit.

The structure: Codex caught the scanner self-reference and the TCP half-close gap; Claude and Gemini flagged the Accept/ReadFrom error-handling termination; Gemini and Claude both caught the partial-failure path; all three agents independently spotted the stale-pending-bindings issue. The non-critical work splits into reconciliation-loop fixes (I2/I3) and forwarder hardening (I1/I4/S1–S5). Test coverage gaps are concentrated in UDP and error-injection paths.

// relayUDP forwards UDP datagrams between the bindIP listener and
// 127.0.0.1, multiplexing concurrent flows by source address. Each
// flow holds a dedicated outbound socket; the socket closes after
// udpIdleTimeout of no upstream reply.

The function returns after the first copy completes; defer in.Close() and defer out.Close() (lines 149, 156) then forcibly close both sockets. A client that calls CloseWrite after sending a request — the standard signal for end-of-request in HTTP/1.x with Connection: close, FTP control, and several custom protocols — makes the in → out copy finish first; the proxy tears down the upstream connection before the response can drain. The pre-PR DNAT preserved TCP half-close because the kernel proxied at the IP layer.

Fix: half-close per direction and wait for both copies before tearing down.

copyDone := func(dst, src net.Conn) {
    _, _ = io.Copy(dst, src)
    if tcp, ok := dst.(*net.TCPConn); ok {
        _ = tcp.CloseWrite()
    }
    done <- struct{}{}
}

// relayUDP forwards UDP datagrams between the bindIP listener and
// 127.0.0.1, multiplexing concurrent flows by source address. Each
// flow holds a dedicated outbound socket; the socket closes after
// udpIdleTimeout of no upstream reply.
func (f *loopbackForwarder) relayUDP(ctx context.Context, pc net.PacketConn, port uint16) {
	target := net.JoinHostPort("127.0.0.1", strconv.Itoa(int(port)))

	var (
		mu    sync.Mutex
		flows = make(map[string]net.Conn)
	)
	defer func() {
		mu.Lock()
		for k, out := range flows {
			_ = out.Close()
			delete(flows, k)
		}
		mu.Unlock()
	}()

	buf := make([]byte, 64*1024)
	for {
		n, src, err := pc.ReadFrom(buf)
		if err != nil {
			return
		}
		srcKey := src.String()

		mu.Lock()
		out, ok := flows[srcKey]
		if !ok {
			conn, dialErr := f.dialer.DialContext(ctx, "udp", target)
			if dialErr != nil {
				mu.Unlock()
				log.Debugf("loopback forwarder dial udp/%d: %s", port, dialErr)
				continue
			}
			out = conn
			flows[srcKey] = out
			go f.readUDPReplies(pc, out, src, srcKey, &mu, flows)
		}
		mu.Unlock()

		_, _ = out.Write(buf[:n])
	}
}

const udpIdleTimeout = 30 * time.Second

// readUDPReplies pumps datagrams from a per-flow outbound socket back
// to the original client. A SetReadDeadline of udpIdleTimeout per Read
// closes the flow once the upstream stops replying.
func (f *loopbackForwarder) readUDPReplies(pc net.PacketConn, out net.Conn, srcAddr net.Addr, srcKey string, mu *sync.Mutex, flows map[string]net.Conn) {
	replyBuf := make([]byte, 64*1024)
	for {
		_ = out.SetReadDeadline(time.Now().Add(udpIdleTimeout))
		n, err := out.Read(replyBuf)
		if err != nil {
			mu.Lock()
			if flows[srcKey] == out {
				delete(flows, srcKey)
			}
			mu.Unlock()
			_ = out.Close()
			return
		}
		_, _ = pc.WriteTo(replyBuf[:n], srcAddr)
	}
}

Design Observations ¶

Concerns ¶

Hand-rolled UDP demultiplexer where the in-tree gvisor-tap-vsock library covers it (in-scope) Claude

S1 covers the code-level swap. As a structural observation, the PR replaces 665+1187 lines of state machine and tests with a 231-line forwarder plus 244 lines of tests. The remaining novelty is concentrated in the UDP flow table — the per-flow goroutine, the mutex passed by pointer to readUDPReplies, the connection-tracking map, the deferred cleanup. forwarder.UDPProxy is the same code Lima uses and is exercised in the gvisor-tap-vsock host-switch process this PR already routes through. Adopting it shrinks the new package by another ~20% and folds S2/S3/S4 into upstream.

Scanner sees its own forwarder; reconciliation needs ownership filtering (in-scope) Codex

C1 covers the symptom. The underlying issue: the new forwarder creates sockets in the same namespace the scanner monitors, so procnet results no longer represent purely external state. Modeling published bindings separately from observed sockets — or filtering forwarder-owned tap-IP sockets out of the scan before the published/pending loops run — eliminates C1 and makes I2 harder to reintroduce. The forwarder already holds the authoritative bindIP/port set in f.tcp and f.udp.

publish treats tracker.Add and forwarder.Add as independent when they form a unit (in-scope) Claude Gemini

I3 covers the symptom. Without the forwarder, the tracker entry misleads host-switch into routing traffic to a missing listener. A publish that returns success only when both steps succeed (with rollback on the second-step failure) lets Tick keep the port in pending and retry next scan. The current "log and continue" pattern is the right shape for stateless rules and the wrong shape for stateful resources.

Strengths ¶

  • Abandoning engine-chain probing entirely (rather than another refinement pass on #10280's heuristic stack) removes the coordination surface that motivated the bug class — a strict reduction in the failure-mode surface area.
  • The Tick(scanned) factoring with injected tracker.Tracker and loopbackController makes the reconciliation logic directly unit-testable in plain Go without iptables or network plumbing.
  • Adding test:unit:guestagent to package.json so the package's tests actually run in CI closes a gap that #10262 and #10280 both rediscovered. Several of the issues above would have been caught by the test cases already present in the package once they ran.
  • Binding the userspace listener on the tap IP integrates cleanly with the existing gvisor-tap-vsock routing — no new path to add or remove from host-switch's perspective.

Review Process Notes ¶

Repo context updates

  • [repo] When reviewing changes to src/go/guestagent/pkg/procnet/scanner_linux.go, check whether the diff introduces any new in-namespace listener (TCP or UDP) opened by the guestagent process itself. procnettcp.ParseFiles reads /proc/net/{tcp,udp} for the entire namespace, so any socket the guestagent opens — userspace forwarders, debug listeners, health checks — appears in the scanned set and will short-circuit the published-port reconciliation loops unless filtered out before Tick. The forwarder's authoritative bindIP/port set is the natural filter source; review against that. Rationale: this is the exact category that produced PR #10357's C1 leak and the bug pattern recurs for any future scanner that observes its own outputs.

Skill improvements

  • When a change introduces a new in-process listener or actor whose outputs are observed by the same component's monitoring loop, trace from the monitor's input source to the new actor's emission point and confirm the actor's outputs are either filtered or distinguishable. State observers that include their own emissions in the observation produce reconciliation loops that never converge for that key. The principle: any change that turns a one-way external observation into a closed-loop self-observation needs an explicit ownership filter, regardless of whether the loop is symptomatic in the test suite.
