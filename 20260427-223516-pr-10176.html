Executive Summary ¶

All nine Round 2 suggestions landed in this commit. The schema / helper plumbing is in good shape: the Moby comment was rewritten, the basename-collision suppression in fetchUpstreamChecksums shipped, the manifest cache is now module-scoped, the populate-dependency-checksums.ts migration script was deleted, the dead checksumAlgorithm plumbing is gone, the lookupChecksum error path now distinguishes "dependency missing" from "artifact missing," RawEntry.checksums is typed optional, and the chartFilename/crdsFilename indentation is back in sync.

The notable new finding is a regression introduced by Round 2's S3 fix itself: download() now hashes existing destination files against the upstream-pinned sha256 before short-circuiting, but call sites that combine expectedChecksum with codesign (DockerCLI on darwin at scripts/dependencies/tools.ts:160-163, DockerProvidedCredHelpers on darwin at scripts/dependencies/tools.ts:489-495) now compare the locally-resigned Mach-O bytes against the upstream pre-signature digest. The post-codesign hash never matches; every macOS postinstall re-downloads and re-signs Docker CLI and the Docker credential helpers, and the postinstall fails offline even when a known-good signed binary is on disk. I1 below proposes a one-line codesign short-circuit.

The rest of the surface produces refinement-grade suggestions: archive helpers (downloadTarGZ / downloadZip) skip the same way Round 2's S3 explicitly scoped out (S1); MobyOpenAPISpec.getAvailableVersions resolves a commit for every historical apiVersion when only the winner is used (S2); a small set of getChecksums enumerations omit windows/arm64 while a few peer enumerations include it (S4); the parseSha256Checksum regex rejects uppercase hex from manual edits (S3); and a handful of polish items (Sha256Checksum return type, diagnostic preservation in downloadAndHash, readDependencyManifest schema validation, GNU regex tightening, MACOS_RUNNER hard-coding, WiX v-prefix asymmetry, in-file no-comments warning).

Verdict: merge with the codesign fix folded in; everything else is non-blocking.

export async function downloadTarGZ(url: string, destPath: string, options: ArchiveDownloadOptions = {}): Promise<string> {
  const overwrite = options.overwrite ?? false;
  const access = options.access ?? fs.constants.X_OK;

  if (!overwrite) {
    try {
      await fs.promises.access(destPath, access);
      console.log(`${ destPath } already exists, not re-downloading.`);

      return destPath;
    } catch (ex: any) {
      if (ex.code !== 'ENOENT') {
        throw ex;
      }
    }
  }
  const binaryBasename = path.basename(destPath, '.exe');
  const workDir = fs.mkdtempSync(path.join(os.tmpdir(), `rd-${ binaryBasename }-`));
  const fileToExtract = options.entryName || path.basename(destPath);

  try {
    const tgzPath = path.join(workDir, `${ binaryBasename }.tar.gz`);
    const args = ['tar', '-zxf', tgzPath, '--directory', workDir, fileToExtract];
    const mode =
            (access & fs.constants.X_OK) ? 0o755 : (access & fs.constants.W_OK) ? 0o644 : 0o444;

    await download(url, tgzPath, { ...options, access: fs.constants.W_OK });
    if (os.platform().startsWith('win')) {
      // On Windows, force use the bundled bsdtar.
      // We may find GNU tar on the path, which looks at the Windows-style path
      // and considers C:\Temp to be a reference to a remote host named `C`.
      const systemRoot = process.env.SystemRoot;

      if (!systemRoot) {
        throw new Error('Could not find system root');
      }
      args[0] = path.join(systemRoot, 'system32', 'tar.exe');
    }
    await simpleSpawn(args[0], args.slice(1));
    await fs.promises.mkdir(path.dirname(destPath), { recursive: true });
    await fs.promises.copyFile(path.join(workDir, fileToExtract), destPath);
    await fs.promises.chmod(destPath, mode);
  } finally {
    fs.rmSync(workDir, { recursive: true, maxRetries: 10 });
  }

  return destPath;
}

/**
 * Returns the stored sha256 for the given artifact as raw hex, with the
 * `sha256:` prefix stripped so it slots straight into `download()`.  Throws

export function lookupChecksum(
  context: DownloadContext,
  name: keyof DependencyVersions,
  artifactName: string,
): string {

S6. downloadAndHash deletes the temp artefact on logical error Claude¶
scripts/lib/dependencies.ts
 * Downloads `url` to a temporary file, hashes it with sha256, and returns the
 * prefixed checksum.  When `options.verify` is provided, also hashes the file
 * with the given algorithm and confirms it matches the expected value;
 * rddepman uses this to cross-check upstream checksum files at bump time.
 */
export async function downloadAndHash(
  url: string,
  options: { verify?: { algorithm: 'sha256' | 'sha512', expected: string | undefined } } = {},
): Promise<Sha256Checksum> {
  const workDir = await fs.promises.mkdtemp(path.join(os.tmpdir(), 'rddepman-'));
  const tempPath = path.join(workDir, 'artifact');

  try {
    await download(url, tempPath, { overwrite: true, access: fs.constants.W_OK });

    if (options.verify) {
      if (!options.verify.expected) {
        throw new Error(
          `No upstream ${ options.verify.algorithm } checksum found for ${ url }. ` +
          `The sidecar checksum file did not list this artifact (filename mismatch, ` +
          `unsupported sidecar format, or missing entry).`,
        );
      }

      const actual = await hashFile(tempPath, options.verify.algorithm);

      if (actual !== options.verify.expected) {
        throw new Error(
          `Upstream checksum mismatch for ${ url }: ` +
          `expected ${ options.verify.algorithm }:${ options.verify.expected }, got ${ options.verify.algorithm }:${ actual }`,
        );
      }
    }

    const sha256 = await hashFile(tempPath, 'sha256');

    return `sha256:${ sha256 }`;
  } finally {
    await fs.promises.rm(workDir, { recursive: true, maxRetries: 10 });
  }
}

Review Process Notes ¶

Skill improvements

  • Add to the review dimensions a sub-bullet under "Backwards compatibility / regressions": When a prior round's fix added a new check, look for paths where the post-write state differs from the as-downloaded state (codesigning, chmod, in-place rewrites). The check sees the post-write bytes and may mismatch a digest recorded over the as-downloaded bytes.

Why this generalises: the rancher-desktop fix exposed the pattern through a single agent (Codex) tracing download() past the cache-hit hash to the trailing codesign call. The underlying shape is broader — any helper that hashes a destination it later mutates will produce the same false-mismatch loop. Common shapes: setuid bits added after install, locale-baked binaries patched in-place, install-time symlink rewriting.
