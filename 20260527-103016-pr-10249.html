S8. Steve.process is never reset to undefined after exit or stop() Claude¶

stop() and the exit handler both zero #port but leave this.process pointing at the dead child. The isRunning getter masks this for callers because it checks exitCode/signalCode, but reading other fields of the stale ChildProcess (e.g., for diagnostics) returns yesterday's values. Clearing this.process = undefined at the same site as this.#port = 0 keeps the two state slots in lockstep.

Design Observations ¶

Concerns

  • Single-channel request/response over IPC for the steve port (future) Claude. pkg/rancher-desktop/store/steve.ts:24-32 and pkg/rancher-desktop/backend/steve.ts:30-35 use one 'steve-port' channel for both directions: the renderer "asks" by sending it with no args, the main process broadcasts the current port back on the same channel. The contract — "send-with-no-args means request; reply arrives via the broadcast" — is implicit and uncommented. Two channels (steve-port-get for the request, steve-port for the broadcast) would make the direction explicit and match the existing IpcMainEvents / IpcRendererEvents type split.
  • Dashboard window lifecycle is independent of Steve lifecycle (in-scope) Codex. I1 is the symptom of a missing rule: nothing closes or reloads the dashboard BrowserWindow when Steve stops. A Steve.on('stop')-style hook that closes (or marks-stale) the dashboard window would close the class of bugs that the per-call URL-check fix only patches one site of.

Strengths

  • Retiring the in-process Express server and the bespoke proxyUtils.ts removes a known-difficult onProxyReqWs path and a non-trivial chunk of IPC surface Claude Codex Gemini.
  • The Round 1 fix package — pendingStart guard, port-not-stored-until-ready, dashboard button gated on port > 0, e2e waitForEvent registered before click — addresses every Round 1 important finding with a clean, narrowly-scoped change Claude Codex.
  • Latch<number> + port-on-stdout + Vuex broadcast is a clean, debuggable replacement for the prior fixed-port /api/steve-port HTTP route Claude Gemini.

Review Process Notes ¶

Skill improvements

  • When a previous round's fix replaces an explicit enumeration with a filter (regex, predicate, allow-list expressed declaratively), check whether the filter still enforces the completeness the enumeration did. A filter accepts what matches; an enumeration asserts what must be present. Add to the Correctness dimension's "Enumeration symmetry" pattern: a regression that converts "iterate required items, check each exists" into "iterate available items, keep matches" silently weakens the required-set guarantee even when every individual filter is correct.
  • When a PR introduces a long-lived window or surface that depends on a backend resource (port, socket, certificate) whose value can change across a stop/start cycle, enumerate every read of that resource across the window's lifecycle — including the restore/reuse path. The bug shape is reliable: the cold-open path picks up the new value, the restore path replays the cached one.
