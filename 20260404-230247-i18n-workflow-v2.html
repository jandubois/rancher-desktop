Executive Summary ¶

This branch adds a comprehensive i18n system to Rancher Desktop: a Go-based i18n-report CLI tool for translation management, main-process and renderer-side translation modules, locale settings integration, and full UI translation into German and Simplified Chinese. The architecture is sound — clean separation between lightweight main-process interpolation and renderer-side ICU formatting, reactive locale switching via Vuex, and a robust translation maintenance pipeline with drift detection, validation, and manifest cross-validation.

The review found 16 important issues and ~25 suggestions across 4 passes. The most impactful issues are: (1) translated validator errors breaking locked-field regex detection, (2) missing raw=true on two v-html + t() calls causing escaped HTML in production, (3) IPC listener leak from non-idempotent store initialization, and (4) several merge tool robustness gaps (duplicate keys, unknown keys, non-mapping root handling). No critical bugs were found — the feature gate limits runtime exposure.

None.

The tag validator compares tag-name counts only. It accepts translations that keep an <a> tag but drop required data-action/data-navigate attributes, and also accepts malformed open/close pairs. The runtime handlers at navigateClick.ts:10-14 and KubernetesError.vue:118-120 depend on these attributes.

src/go/i18n-report/report_validate.go

// tagRe matches HTML-like tags: <tag>, </tag>, <tag/>, <tag attr="...">.

// checkTags verifies that the locale value contains the same HTML-like
// tag names as the English source.
func checkTags(key, enValue, localeValue string) []validationError {
	enTags := extractTagNames(enValue)
	localeTags := extractTagNames(localeValue)

	var errors []validationError

	for tag, enCount := range enTags {
		localeCount := localeTags[tag]
		if localeCount < enCount {
			errors = append(errors, validationError{
				Key:     key,
				Check:   "tag",
				Message: fmt.Sprintf("missing <%s> tag (expected %d, found %d)", tag, enCount, localeCount),
			})
		}
	}

Fix: Compare (tag name, direction, required attributes) instead of just name counts.

// reportValidateQuiet runs validate and returns an error summary without
// printing individual errors.
func reportValidateQuiet(root, locale string) error {
	enPath := translationsPath(root, "en-us.yaml")
	localePath := translationsPath(root, locale+".yaml")

	enKeys, err := loadYAMLFlat(enPath)
	if err != nil {
		return err
	}
	localeKeys, err := loadYAMLFlat(localePath)
	if err != nil {
		return err
	}

	doc, err := loadYAMLDocument(localePath)
	if err != nil {
		return err
	}

	var errorCount int

	// Manifest consistency.
	if m, err := loadManifest(root); err != nil {
		errorCount++
	} else if _, registered := m.Locales[locale]; !registered {
		errorCount++
	}

	// Placeholder, ICU, and tag checks.
	for key, enValue := range enKeys {
		localeValue, exists := localeKeys[key]
		if !exists {
			continue
		}
		errorCount += len(checkPlaceholders(key, enValue, localeValue))
		errorCount += len(checkICUStructure(key, enValue, localeValue))
		errorCount += len(checkTags(key, enValue, localeValue))
	}

	// Override placement.
	errorCount += len(validateOverridePlacement(doc))

	if errorCount > 0 {
		return fmt.Errorf("%d validation errors", errorCount)
	}
	return nil
}

15. merge writes unknown/hallucinated keys Codex ¶

The parser checks syntax only via isValidDottedKey(); it never verifies that the key exists in en-us.yaml. An LLM typo or hallucinated key enters the locale file without metadata, undermining the drift machinery.

16. merge can erase locale file on non-mapping YAML root Codex ¶

If a locale file has a scalar or sequence root, nodeSetLeaf() mutates the non-mapping node, but serializeYAMLNode() refuses to emit anything for non-mapping roots. The subsequent os.WriteFile() then truncates the locale file to empty.

src/go/i18n-report/report_merge.go

	localePath := translationsPath(root, locale+".yaml")

	// Load existing YAML as a node tree (empty tree if file does not exist).
	doc, err := loadYAMLDocument(localePath)
	if err != nil {
		return fmt.Errorf("loading existing %s: %w", localePath, err)
	}
	treeRoot := documentRoot(doc)

	// Build input reader from file arguments or stdin.
	var inputReader io.Reader
	if len(files) > 0 {
		var combined strings.Builder
		for _, path := range files {
			data, err := os.ReadFile(path)
			if err != nil {
				return fmt.Errorf("reading %s: %w", path, err)
			}
			combined.WriteString(extractTranslationText(data))
			combined.WriteString("\n")
		}
		inputReader = strings.NewReader(combined.String())
	} else {
		inputReader = os.Stdin
	}

Design Observations ¶

Concerns ¶

D1. Translated error strings as machine-readable identifiers (in-scope) Claude Codex Gemini ¶

The fieldLocked pattern is structural: validation produces translated human-readable strings, and callers regex-match them to determine error types. A { code: string, message: string } return type would eliminate this class of bugs.

D2. Reference scanner as regex inventory (future) Codex ¶

The scanner is a list of "known syntaxes," which is why it missed v-t immediately. A syntax table or small parser layer would reduce maintenance burden as new call patterns emerge.

D3. data-navigate in translation strings couples navigation to content (in-scope) Claude ¶

Translators must preserve exact page names and tab IDs inside HTML attributes. Extracting navigation targets into component props would eliminate this fragility.

D4. Manifest "source locale" abstraction half-integrated (future) Codex ¶

manifest.go models an arbitrary source locale, but most commands hardcode en-us.yaml.

Strengths ¶

  1. pendingLocale cancellation pattern in main/i18n.ts elegantly handles rapid locale changes within Node's single-threaded event loop. Claude
  2. onLocaleChange callback API cleanly separates locale loading from menu/tray rebuilding, with proper unregister support. Claude
  3. Test mock design — testUtils/translations.ts loads real English YAML and performs real interpolation, avoiding mocks that test themselves. Claude
  4. Reactive locale switching is well-architected: page titles use titleKey via Vuex, preference tabs use labelKey, nav items use routeLabelKeys. DiagnosticsBody.vue headers correctly moved to computed. Claude
  5. Manifest + metadata + drift-detection model turns translation freshness into something the repo can validate instead of relying on manual review. Codex
  6. Two-tier i18n architecture (simple interpolation in main, ICU MessageFormat in renderer) keeps Electron main free of unnecessary complexity while sharing YAML sources. Claude Codex Gemini
  7. yaml.Node-based merge preserves comments and leaf structure directly in the tree, which is the right fit for translation maintenance. Codex
