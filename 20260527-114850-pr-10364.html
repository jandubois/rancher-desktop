Executive Summary ¶

The PR introduces a runningDistros getter (wsl --list --running --quiet) and uses it to filter sync() and getStateForIntegration() so stopped distros are not started just to query them. The fix is well-scoped, but it leaves three gaps: enabled-but-stopped distros are still started during initial sync when the keepalive proxy cannot run yet (I1), the verifyAllDistrosKubeConfig diagnostic still execs into stopped distros (I2), and listIntegrations runs wsl --list --running once per distro instead of once per call (I3).

Design Observations ¶

Concerns

  • Property getters that execute subprocesses (in-scope) Gemini Claude: runningDistros (and the pre-existing nonBlacklistedDistros) are property getters that fork wsl.exe. Property access reads as cheap and side-effect-free, but each await this.runningDistros spawns a process and there is no memoization. The N+1-invocation pattern flagged in I3 is the direct consequence. Refactoring to explicit async getRunningDistros() (or a memoize-per-sync wrapper) would make the cost visible at call sites and remove an entire class of accidental fan-out bugs.
  • One of three exec-into-distro call sites left unguarded (in-scope) Claude: The PR fixes sync() and getStateForIntegration() but not verifyAllDistrosKubeConfig (I2). Closing the whole class in this PR is a few lines and keeps the fix coherent.
  • Race between the running-distros check and the exec (future): After runningDistros resolves, a distro can idle-terminate before syncDistro reaches it. The window is small but non-zero. A retry-or-skip path when the subsequent wsl --exec discovers the distro stopped would close it.
  • The "trust settings for stopped distros" path is silent (future) Claude: A console.debug breadcrumb would help operators distinguish "marker said X" from "we did not read the marker because the distro was stopped" when triaging integration logs.

Strengths

  • wsl --list --running --quiet is the right primitive — observes state without creating a session in the target distro Codex.
  • runningDistros returns an empty set on error rather than throwing, which keeps the legacy "consider all distros" behavior when the listing itself fails. Defaulting in the safe direction is right (see S5 for the related observability gap) Claude.
  • The new tests mock by argument matching (if (args.includes('--running'))) instead of re-mocking the whole module, which preserves the existing nonBlacklistedDistros/supportedDistros tests unchanged Claude.

Skill improvements

  • When a PR adds a getter that performs subprocess I/O, audit every call site for unintended fan-out. JavaScript/TypeScript property getters look free at the call site, so a getter inside a .map(...) body issues one subprocess per element rather than one per call. Flag this even when the diff only adds the getter and one obvious call site — the cost surfaces at unrelated callers (here, listIntegrations) that the getter does not name.
