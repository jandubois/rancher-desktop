Executive Summary ¶

This branch adds comprehensive i18n support to Rancher Desktop: a Go maintenance CLI (i18n-report), main-process and renderer translation infrastructure, a locale setting with language picker, and translated strings across ~40 Vue components. The architecture is well-designed, the commit history is clean, and the feature gate is pragmatic. The review found one navigation regression (broken tab-level data-navigate), several data-integrity bugs in the Go tooling (multiline handling, metadata parsing, missing manifest entry), a locale-revert gap in the main process, and various moderate issues around initialization ordering and translation-string safety.

I-5. Multiline metadata truncation causes false drift Codex ¶

writeMetadata() emits multiline English strings as YAML block scalars, but parseMetadata() line-splits and captures only the first line (e.g., |-). When reportDrift() compares the full English value against this truncated metadata, untouched multiline keys are always flagged as drifted. Running drift --locale=de falsely reports five drifted keys.

src/go/i18n-report/metadata.go
// parseMetadata parses flat "dotted.key: value" lines into a map.
// Values may be YAML-quoted; quotes are stripped.
func parseMetadata(content string) (map[string]string, error) {
	result := make(map[string]string)
	for _, line := range strings.Split(content, "\n") {
		line = strings.TrimSpace(line)
		if line == "" || strings.HasPrefix(line, "#") {
			continue
		}
		idx := strings.Index(line, ": ")
		if idx < 0 {
			continue
		}
		key := line[:idx]
		value := stripYAMLQuotes(line[idx+2:])
		result[key] = value
	}
	return result, nil
}

// source to find translated keys whose English text has changed.
func reportDrift(root, locale string) error {

	localePath := translationsPath(root, locale+".yaml")

	enKeys, err := loadYAMLFlat(enPath)
	if err != nil {
		return err
	}

	if err != nil {
		return err
	}

	localeLeaves := nodeAllLeaves(treeRoot)

	meta, err := loadMetadata(root, locale)
	if err != nil {
		return err
	}

// loadYAMLDocument loads a YAML file into a yaml.Node document tree.
// Returns a DocumentNode wrapping a MappingNode root.
// If the file does not exist, returns an empty document.
func loadYAMLDocument(path string) (*yaml.Node, error) {
	data, err := os.ReadFile(path)
	if os.IsNotExist(err) {
		// Return an empty document with an empty mapping root.
		return &yaml.Node{
			Kind: yaml.DocumentNode,
			Content: []*yaml.Node{
				{Kind: yaml.MappingNode},
			},
		}, nil
	}
	if err != nil {
		return nil, err
	}

Suggestions ¶

S-1. XSS risk from v-html rendering of translation strings Claude ¶

Translation values rendered via v-html could inject arbitrary HTML. The YAML files are generated by AI agents and merged without HTML sanitization. In Electron with nodeIntegration: true, XSS is equivalent to arbitrary code execution. Practically, the attack requires compromising translation files built into the app at compile time, so the risk is low. Defense-in-depth: add a validation check in report_validate.go that rejects <script, on[a-z]+=, and javascript: in values rendered via v-html.

S-2. Nested ICU validation gap Gemini ¶

extractPlaceholderNames only extracts placeholders at brace depth == 0, missing nested placeholders inside ICU branches (e.g., {name} inside {count, plural, other {items for {name}}}). Document the limitation or replace with an AST-based ICU parser.

src/go/i18n-report/report_validate.go
// extractPlaceholderNames returns the set of top-level placeholder variable
// names from a translation string. Only braces at depth 0 are considered
// placeholders; nested braces (ICU plural/select branches) are skipped.
func extractPlaceholderNames(s string) map[string]bool {
	result := make(map[string]bool)
	depth := 0
	for i := 0; i < len(s); i++ {
		switch s[i] {
		case '{':
			if depth == 0 {
				// Extract the identifier starting after '{'.
				rest := s[i+1:]
				if m := identRe.FindStringSubmatch(rest); m != nil {
					result[m[1]] = true
				}
			}
			depth++
		case '}':
			if depth > 0 {
				depth--
			}
		}
	}
	return result
}
S-3. ICU branch parser ignores quoted braces Gemini ¶

The brace-counting logic does not account for ICU literals wrapped in single quotes (e.g., '{'), causing depth tracking to desync on strings with literal braces.

S-4. Iterative string replacement allows placeholder cross-contamination Gemini ¶

Sequential replaceAll calls mean an earlier argument's value could contain a later argument's placeholder pattern. Unlikely in practice, but a single regex replace pass would be safer.

pkg/rancher-desktop/main/i18n.ts
export function t(key: string, args?: Record<string, string | number>): string {
  let msg = getByPath(translations[currentLocale], key)
         ?? getByPath(translations['en-us'], key);

  if (msg === undefined) {
    return `%${ key }%`;
  }

  if (args) {
    for (const [name, value] of Object.entries(args)) {
      msg = msg.replaceAll(`{${ name }}`, String(value));
    }
  }

  return msg;
}

Design Observations ¶

Concerns ¶

HTML in translation strings is fragile (in-scope) Claude: Five translation keys embed <a> tags with data-navigate or data-action attributes that application code relies on for click handling. Translators can easily break navigation by altering these attributes. Using named slots or structured placeholders (e.g., {link_start}WebAssembly{link_end}) would let the code own the HTML while translators handle only text, and would eliminate the v-html usage.

Main-process t() used in API error messages (future) Claude: Validation errors from settingsValidator.ts flow through the HTTP command API. Translating API error messages means rdctl output language depends on the server's locale, not the CLI user's preference. Consider keeping API-facing errors in English.

Three coupled artifacts per locale (in-scope) Codex: The tool now maintains locale YAML, metadata YAML, and manifest registration per locale. Only merge updates locale state holistically; remove, meta, validate, check, and manifest each touch a subset. A shared "load/mutate/rewrite locale+metadata/validate manifest" layer would remove this split-brain state.

Dual interpolation systems (in-scope) Claude: The main process uses {variable} while the renderer uses ICU MessageFormat. Translation strings must use different syntax depending on which process renders them. The validate command does not distinguish between the two schemes.

Strengths ¶

  • The feature gate (application.locale = 'none' hiding the selector) is pragmatic — it ships infrastructure without committing to translation quality.
  • The onLocaleChange callback pattern in main/i18n.ts cleanly decouples locale loading from menu/tray rebuilding. The pendingLocale guard correctly handles rapid locale switches.
  • The Go i18n-report tool is well-designed with comprehensive validation (placeholder parity, ICU structure, tag matching, metadata coherence).
  • The titleKey/descriptionKey approach for reactive page headers avoids re-rendering entire pages while keeping titles in sync.
  • Using import.meta.webpackContext for locale loading keeps the approach consistent with the existing build pipeline.
