## Open issues

### High — `curl | bash` equivalents and self-fetches from a moving ref

- **`scripts/windows-setup.ps1:49`** — `Invoke-WebRequest -UseBasicParsing -Uri 'https://get.scoop.sh' | Invoke-Expression`. PowerShell equivalent of `curl … | sh`; whoever controls `get.scoop.sh` gets arbitrary code execution on the developer machine. Pin the URL to a 40-hex commit on the scoop installer repo, download to disk, verify a stored sha256, then execute. (See wiki §3.5; PowerShell threat model identical to shell.)
- **`scripts/windows-setup.ps1:74`** — fetches `install-wsl.ps1`, `restart-helpers.ps1`, `sudo-install-wsl.ps1`, `uninstall-wsl.ps1` from `https://raw.githubusercontent.com/rancher-sandbox/rancher-desktop/main/scripts/windows/$file`. The `main` branch is mutable; anyone with push access controls these bytes. Replace with a relative path (these files live next to the caller) or pin to a 40-hex SHA.

### Medium — unverified inline downloads (version-pinned, no digest)

- **`resources/setup-spin:38-43`** — `spin templates install --update --tar https://github.com/spinframework/${repo}/archive/refs/tags/${SPIN_TEMPLATES_TAG}.tar.gz`, with a fallback to `…/refs/heads/main.tar.gz` on failure. The pinned-tag→fallback-to-main pattern actively undermines the pinning — a transient network error or a tag deletion silently degrades to the unpinned path. Drop the `main` fallback; make tag retrieval failure fatal. Track the tarball sha256 in `dependencies.yaml` (the `spinKubePlugin` entry already has an empty `checksums: {}` block ready to populate).
- **`resources/setup-spin:53-59`** — `spin plugins install --yes --url "https://github.com/spinframework/spin-plugins/raw/refs/heads/main/manifests/${plugin}/${plugin}@${version}.json"`, with a fallback to the unversioned `${plugin}.json`. Branch-pinned to `main`; mutable. The plugin manifest itself dictates what binary `spin plugins install` downloads, so compromise of the manifest is full RCE on the user's machine. Fetch from a tagged release URL or vendor the manifest bytes; verify a digest before passing the URL to `spin plugins install`.
- **`scripts/windows/install-wsl.ps1:39`** — `Invoke-WebRequest -Uri https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi -OutFile $wslMsiFile`; `msiexec /i$wslMsiFile`. MSI is run with admin privileges with no signature/digest check beyond Windows' built-in code-signing. Either record a sha256 in `dependencies.yaml` and verify before `msiexec`, or assert `Get-AuthenticodeSignature` reports `Valid` for a Microsoft signing chain.
- **`.github/actions/setup-environment/action.yaml:45-46`** — `curl --location --output "$bindir/yq.exe" https://github.com/mikefarah/yq/releases/download/v4.43.1/yq_windows_amd64.exe`. Version pinned, no digest. Pin a sha256 (yq publishes a `checksums.txt` sidecar at the same release) and `sha256sum -c` after download, or move yq to `dependencies.yaml`.
- **`bats/Makefile:39, 44`** — `wget --no-verbose $(JQ_URL)/jq-osx-amd64`, `wget --no-verbose $(JQ_URL)/jq-linux64`. `JQ_VERSION` defaults to `1.7.1`. jq publishes a sha256 sidecar (`sha256sum.txt`); add a `sha256sum --check` step after each `wget`.
- **`.github/workflows/test.yaml:20`** — `go install mvdan.cc/sh/v3/cmd/shfmt@latest`. Floating `@latest`; even though Go module proxy hash-verifies on fetch, the version itself is unpinned. Pin a concrete version (`@v3.x.y`) or add `shfmt` to `dependencies.yaml` so `rddepman` keeps it current.
- **`bats/scripts/ghcr-mirror.sh:43`** + **`bats/tests/helpers/images.bash:11`** — `IMAGES=(alpine busybox nginx python python:3.9-slim ruby tonistiigi/binfmt registry:2.8.1)` — most entries are bare image names (floating `:latest`); `skopeo copy --all "docker://$IMAGE" "docker://$GHCR_REPO/$IMAGE"`. The mirror runs ad-hoc, so the tests subsequently consume whatever was on Docker Hub at mirror time. Pin every entry to `image:tag@sha256:<digest>`; `skopeo` copies digests transparently.
- **`scripts/spelling.sh:66`** — `git -c advice.detachedHead=false clone --branch "$version" --depth 1 "$repo" "$checkout"` against `https://github.com/check-spelling/check-spelling`. Tag-pinned, not SHA-pinned; `--depth 1 --branch <tag>` makes a later upgrade-to-SHA require a re-clone. Replace `$version` with a 40-hex SHA from `dependencies.yaml` (the `check-spelling` entry has an empty `checksums: {}` block — populate it with the commit SHA, e.g. as a sidecar `commit:` field).
- **`.github/workflows/smoke-test.yaml:66`** — `curl -L -o rancher-desktop.AppImage "${OBS_DOWNLOAD_URL}${artifact_name}"`. The next line is `sha512sum rancher-desktop.AppImage > rancher-desktop.AppImage.sha512sum` — a self-hash, not authentication. The artifact name is queried from the OBS index seconds before; an attacker who can swap the index can swap the binary. Either generate the sha512sum at OBS publish time and verify it here, or sign the AppImage and verify with `cosign verify-blob` / `gpg --verify`.

- **Self-fetch of own release zip** — `.github/workflows/linux-release.yaml:36`: `curl -L -o "dist/${release_zip_name}" "https://github.com/${repository}/releases/download/${version_with_v}/${release_zip_name}"`. Self-fetch from a tag-pinned URL; the bytes are the release we just published. Adding a sha512sum sidecar verification would be defense-in-depth against an attacker who can replace release assets without changing the tag.
- **`gh release download` without verifier** — `.github/workflows/smoke-test.yaml:48-55`: downloads `*.dmg`, `*.msi`, `rancher-desktop-linux-*.zip` and their `.sha512sum` sidecars but never runs `sha512sum -c`. Acceptable in CI because dnf authenticates the bytes regardless; flagged for completeness.
- **Insecure transport — local dev plugins server** — `scripts/dev.ts:67`: `agent: new https.Agent({ rejectUnauthorized: false })`. Used only when `RD_ENV_PLUGINS_DEV` is set, against `https://localhost:8888/home`. Localhost-scoped — accept this and document the constraint, or fix the dev cert chain.

- **Custom manifest delegate paths surface as the two empty-checksum issues above** — `golangci-lint` (verified via `go run …@v${ver}` and `scripts/go.sum`) is fine; the other two need fixing as described.

## Retro: dependency types not explicitly covered

R1. **PowerShell `IWR | iex` with multi-line continuation goes undetected.** The script's `curl_bash_equivalents[]` is empty even though `scripts/windows-setup.ps1:49-50` clearly contains the pattern:

```powershell
Invoke-WebRequest -UseBasicParsing -Uri 'https://get.scoop.sh' `
    | Invoke-Expression
```

The trailing backtick splits the pipeline across two lines; the script's regex appears to require both halves on the same line. **Proposed change:** before pattern matching, concatenate lines whose previous line ends in `` ` `` (PowerShell line-continuation), then run the existing detector. Alternatively, add a second pass that finds `Invoke-WebRequest`/`iwr` references and looks ahead up to ~3 lines for a `| Invoke-Expression`/`| iex` consumer.

R2. **Yarn-berry `conditions:` filtering treated as missing-integrity.** The script reports 87 descriptors as `missing_integrity` in `yarn.lock`, but every one of them is a platform-specific optional dep gated by `conditions: os=X & cpu=Y` and never installed on the host platform — yarn-berry intentionally omits the `checksum:` line in that case. **Proposed change:** when walking yarn-berry blocks, skip entries whose body contains `conditions:`. Optionally surface them under a separate `platform_conditioned: <count>` key so the report can mention them without flagging.

R3. **Custom-manifest no-op delegates aren't traced automatically.** The script reports `empty_checksum_blocks: 3` and lists the entry names, but stops there. In this scan I had to grep manually (`scripts/dependencies/tools.ts`) to find that the three entries delegate to `go run`, `scripts/spelling.sh`, and `resources/setup-spin`. **Proposed change:** when an entry name has an empty checksum block, look for a same-name class in the repo's dependency-management directory (`scripts/dependencies/*.ts` here, but configurable) and surface the `download()` body — when it is `Promise.resolve()`, find string literals that look like script paths (`scripts/*.sh`, `resources/*`) and emit them as `delegate: <path>` so Phase 2 can inline-download-rule them straight away.
