Executive Summary ¶

This PR adds i18n-report, a 6,625-line Go CLI tool with 14 subcommands for maintaining Rancher Desktop's translation files. The code is clean, well-tested, and the AST-level YAML manipulation design is a strong foundation for preserving translator annotations through merge operations. No critical issues were found. Four important issues should be addressed before merge: tag validation produces false positives on angle-bracket placeholders in translation strings, the drift check silently passes when metadata is unreadable, locale commands fail on non-existent locale files instead of treating them as empty, and the remove command incorrectly attempts metadata cleanup for the source locale.

I1. Tag validation false positives on angle-bracket placeholders — Codex ¶

The tagRe regex matches angle-bracket placeholders like <ingress-name> as HTML tag ingress (since \w+ captures up to the hyphen). The en-us.yaml string at key ingress-ip-domain contains <ingress-name>.<namespace-name>.<ip address of ingress controller>, which produces synthetic tag names ingress, namespace, and ip. Translations that omit these non-HTML markers trigger false validation errors.

src/go/i18n-report/report_validate.go:222-256
var tagRe = regexp.MustCompile(`</?(\w+)[^>]*/?>`)

// attrRe matches HTML attributes of the form name="value" or name='value'.
var attrRe = regexp.MustCompile(`(data-\w+)=(?:"([^"]*)"|'([^']*)')`)

// checkTags verifies that the locale value contains the same HTML-like
// tag names as the English source, and that required data-* attributes
// are preserved (runtime handlers depend on these).
func checkTags(key, enValue, localeValue string) []validationError {
	enTags := extractTagNames(enValue)
	localeTags := extractTagNames(localeValue)

	var errors []validationError

	for tag, enCount := range enTags {
		localeCount := localeTags[tag]
		if localeCount < enCount {
			errors = append(errors, validationError{
				Key:     key,
				Check:   "tag",
				Message: fmt.Sprintf("missing <%s> tag (expected %d, found %d)", tag, enCount, localeCount),
			})
		}
	}

	for tag, localeCount := range localeTags {
		enCount := enTags[tag]
		if localeCount > enCount {
			errors = append(errors, validationError{
				Key:     key,
				Check:   "tag",
				Message: fmt.Sprintf("unexpected <%s> tag (expected %d, found %d)", tag, enCount, localeCount),
			})
		}
	}

Fix: restrict tag matching to a known set of inline HTML tags (a, b, br, code, em, i, strong, span, p, li, ul, ol), or exclude tokens containing hyphens or spaces from tag matching.

src/go/i18n-report/yaml.go:36-47
// loadYAMLFlat loads a YAML file and returns flattened key-value pairs.
func loadYAMLFlat(path string) (map[string]string, error) {
	data, err := os.ReadFile(path)
	if err != nil {
		return nil, err
	}
	var raw map[string]interface{}
	if err := yaml.Unmarshal(data, &raw); err != nil {
		return nil, fmt.Errorf("parsing %s: %w", path, err)
	}
	return flattenYAML("", raw), nil
}

     data, err := os.ReadFile(path)
+    if os.IsNotExist(err) {
+        return make(map[string]string), nil

Design Observations ¶

Strengths ¶

  • AST-level YAML manipulation — operating on yaml.v3 node trees instead of marshaling through Go structs preserves comments, annotations, and formatting through merge operations. This is the right call for a translation workflow tool. [Claude Codex Gemini]
  • Three-format auto-detection in extractTranslationText — the JSONL → markdown fence → raw text pipeline handles realistic translator output formats without pre-processing. [Claude]
  • Policy-aware checking — the experimental vs shipping policy split maps well to the locale lifecycle, enforcing incrementally stricter requirements as a locale matures. [Claude Gemini]
  • Metadata-based drift detection — storing the English source text at translation time (rather than just a hash) enables meaningful diff output and is more debuggable. [Claude]
  • Sorted insertion — nodeInsertSorted keeps YAML output deterministic regardless of insertion order, important for clean diffs in version control. [Claude]

Concerns ¶

  • (future) loadYAMLFlat and loadYAMLDocument have subtly different semantics for missing files and value representation. loadYAMLFlat unmarshals through map[string]interface{} (where YAML booleans/numbers become Go types), while the Node tree preserves raw strings. A unified loading abstraction would prevent future divergence. [Claude]
